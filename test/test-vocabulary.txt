- Jest 테스트 메서드
describe()	            테스트 그룹을 정의하는 메서드. 여러 개의 it()을 포함할 수 있음.
it()	                단일 테스트 케이스를 정의하는 메서드. 특정 기능이 예상대로 동작하는지 검증.
test()	                it()과 동일한 기능을 제공하지만, it()이 일반적으로 더 많이 사용됨.


-  Jest 단언(Assertion) 메서드
expect(value)	        테스트의 예상값을 정의하는 기본 메서드.
toBe(value)	            === 연산자로 값이 정확히 일치하는지 검증.
toEqual(value)	        객체나 배열이 같은 값인지 비교 (깊은 비교).
toHaveLength(number)	배열이나 문자열의 길이가 특정 값과 같은지 검증.
toContain(value)	    배열이나 문자열이 특정 값을 포함하는지 검증.
toBeTruthy()	        값이 true로 평가되는지 검증.
toBeFalsy()	            값이 false로 평가되는지 검증.
toThrow()	            함수 실행 시 예외가 발생하는지 검증.
rejects.toThrow()	    비동기 함수 실행 시 예외가 발생하는지 검증.

Jest before 및 after 메서드
beforeAll(fn)	    모든 테스트 전에 한 번만 실행. (DB 연결 등)
beforeEach(fn)	    각 테스트 실행 전에 실행. (초기화 작업)
afterEach(fn)	    각 테스트 실행 후 실행. (리소스 정리)
afterAll(fn)	    모든 테스트 후 한 번 실행. (DB 연결 해제 등)


Mock 함수란? (Mock Function)
Mock 함수란, 실제 함수를 실행하지 않고 가짜(Mock)로 대체하여 테스트하는 함수를 의미


- Jest Mocking 관련 용어
jest.fn()	                가짜(Mock) 함수를 생성하는 기본 메서드.
jest.spyOn(object, method)	특정 객체의 메서드를 감시(Spy)하여 호출 여부를 확인.
mockReturnValue(value)	    Mock 함수가 특정 값을 반환하도록 설정.
mockResolvedValue(value)	비동기(Mock Promise) 함수가 특정 값을 반환하도록 설정.
mockRejectedValue(error)	비동기(Mock Promise) 함수가 특정 에러를 발생시키도록 설정.


- Mock 함수 호출 검증
toHaveBeenCalled()	            특정 Mock 함수가 호출되었는지 검증.
toHaveBeenCalledTimes(number)	특정 Mock 함수가 정확히 n번 호출되었는지 검증.
toHaveBeenCalledWith(...args)	특정 Mock 함수가 특정 인자와 함께 호출되었는지 검증.


- 테스트 실행 관련 명령어
npm run test	                   모든 단위 테스트 실행.
npm run test -- <파일경로>	        특정 테스트 파일만 실행.
npm run test -- -t "테스트 이름"    특정 테스트 케이스(describe() 또는 it())만 실행.
npm run test:watch	               파일 변경 시 자동으로 테스트 실행.
npm run test -- --clearCache	   Jest 캐시를 삭제하고 다시 실행.
